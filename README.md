# NFT-инфо бот (Telegram, aiogram)

## Что делает
Пользователь присылает боту ссылку вида:
https://t.me/nft/HomemadeCake-59562

Бот скачивает эту страницу, вытаскивает модель / фон / узор / владельца
и красиво отвечает в чат.

## Установка
```bash
pip install -r requirements.txt
```

## Настройка
1. Получи токен бота у @BotFather в Telegram.
2. Вставь его в `bot.py` в переменную `BOT_TOKEN`.

## Запуск
```bash
python bot.py
```

## Про счётчик "интересовались" и историю владения

Публичная страница t.me/nft/... НЕ содержит счётчик интереса и полную
историю владения — Telegram их просто не публикует. Поэтому бот
собирает эту статистику сам, в файле nft_stats.db (SQLite):
- каждый запрос к конкретному NFT увеличивает счётчик "интересовались"
- если текущий владелец отличается от последнего сохранённого — в
  историю добавляется новая запись с датой

Из-за этого:
- история будет копиться только с момента запуска твоего бота
  (прошлые владения, которые были раньше, узнать неоткуда)
- на БЕСПЛАТНОМ плане Render диск не постоянный — при каждом передеплое
  или перезапуске сервиса файл nft_stats.db обнуляется, и статистика
  начнёт копиться заново. Чтобы это исправить, нужен платный план
  с Persistent Disk (Render Dashboard -> Disks) или внешняя БД
  (например, бесплатный tier Postgres на Render/Supabase).

## Деплой на Render

1. Залей эту папку в свой репозиторий на GitHub (или GitLab).
2. На render.com: New -> Web Service -> подключи репозиторий.
3. Render должен сам подхватить `render.yaml`. Если нет — укажи вручную:
   - Build command: `pip install -r requirements.txt`
   - Start command: `python bot.py`
4. В разделе Environment добавь переменную `BOT_TOKEN` = токен от @BotFather.
5. Задеплой. В логах должно появиться "Bot is running" / старт поллинга.

Бот работает через polling, но чтобы бесплатный Web Service на Render
не считал деплой неудачным (там нужно слушать порт), в bot.py поднят
маленький http-сервер-заглушка на переменной окружения PORT.

Учти: на бесплатном плане Render "усыпляет" сервис при простое —
для бота, который должен отвечать всегда, лучше платный план
или отдельный "keep-alive" пинг (например, через UptimeRobot на корневой URL).

## Статистика: "интересовались" и история владения

Этих данных нет на самой странице t.me/nft/... — Telegram их там не публикует.
Бот сам ведёт учёт в локальном файле `nft_stats.db` (SQLite, создаётся автоматически):

- при каждом запросе конкретного подарка увеличивается счётчик "Интересовались"
- если текущий владелец (спарсенный со страницы) отличается от последнего
  сохранённого — добавляется запись в историю владения с датой и временем

Важные ограничения:
- История считается только с момента, как бот начал использоваться —
  прошлые смены владельца, случившиеся до первого запроса боту, узнать неоткуда.
- На **бесплатном** плане Render диск не постоянный: при каждом передеплое
  или перезапуске сервиса файл `nft_stats.db` обнуляется, и вся накопленная
  статистика теряется. Чтобы данные сохранялись между деплоями, нужен
  платный план Render с постоянным диском (Render Disks), либо внешняя
  БД (например, бесплатный Postgres на Neon/Supabase).
