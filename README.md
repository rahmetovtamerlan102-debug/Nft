# NFT-инфо бот (Telegram, aiogram)

## Что делает
Пользователь присылает боту ссылку вида:
https://t.me/nft/HomemadeCake-59562

Бот скачивает эту страницу, вытаскивает модель / фон / узор / владельца
и красиво отвечает в чат.

## Установка
```bash
pip install -r requirements.txt
```

## Настройка
1. Получи токен бота у @BotFather в Telegram.
2. Вставь его в `bot.py` в переменную `BOT_TOKEN`.

## Запуск
```bash
python bot.py
```

## ВАЖНО: парсер нужно проверить и, скорее всего, подправить
Я написал parser.py по логике "Telegram отдаёт og:description с текстом вида
'Модель: X (1.5%), Фон: Y (2.0%), Узор: Z (0.3%)'". Но у меня в этой среде
нет доступа в интернет, так что я не смог скачать реальную страницу
t.me/nft/... и посмотреть точный HTML.

Если после запуска бот пишет "не удалось распознать поля" — сделай так:
1. Временно добавь в bot.py вывод info.raw_description пользователю
   (или просто напечатай его в консоль через print(info.raw_description)).
2. Пришли мне этот текст (или просто содержимое тега <meta property="og:description">
   со страницы t.me/nft/HomemadeCake-59562, которое можно посмотреть
   через "Просмотр кода страницы" в браузере).
3. Я подгоню регулярки в extract_fields() под точный формат.

Также возможно, что нужные данные (история владения, точные проценты)
подгружаются через JS уже после загрузки страницы (Telegram Web App),
а не лежат в исходном HTML. В этом случае придётся использовать
безголовый браузер (Playwright/Selenium) вместо requests — дай знать,
если так и окажется, я перепишу parser.py под это.

## Деплой на Render

1. Залей эту папку в свой репозиторий на GitHub (или GitLab).
2. На render.com: New -> Web Service -> подключи репозиторий.
3. Render должен сам подхватить `render.yaml`. Если нет — укажи вручную:
   - Build command: `pip install -r requirements.txt`
   - Start command: `python bot.py`
4. В разделе Environment добавь переменную `BOT_TOKEN` = токен от @BotFather.
5. Задеплой. В логах должно появиться "Bot is running" / старт поллинга.

Бот работает через polling, но чтобы бесплатный Web Service на Render
не считал деплой неудачным (там нужно слушать порт), в bot.py поднят
маленький http-сервер-заглушка на переменной окружения PORT.

Учти: на бесплатном плане Render "усыпляет" сервис при простое —
для бота, который должен отвечать всегда, лучше платный план
или отдельный "keep-alive" пинг (например, через UptimeRobot на корневой URL).
